## 📌 Project Overview

Built a **fully dynamic world map dashboard** combining sociodemographic data (population, area, languages, currencies, capitals) with **real-time weather data** (temperature, wind speed, wind direction) for 250+ countries — all fetched live from public REST APIs.

**Key questions:**
- Which countries and regions are the most populated or the largest in area?
- What are the current weather conditions per country in real time?
- How do metrics like temperature and wind speed vary across continents and languages?
- What is the full sociodemographic and meteorological profile of any country on hover?

## 🌐 Data Sources & API Architecture

### API 1 — RestCountries (Sociodemographic Data)

Fields needed: `name`, `cca2`, `ccn3`, `cca3`, `unMember`, `currencies`, `capital`, `region`, `languages`, `latlng`, `area`, `flag`, `maps`, `population`, `continents`

Single endpoint call failed — response too large, Power BI returns 400 Bad Request. Split into two targeted calls:

```
Endpoint 1:
https://restcountries.com/v3.1/all?fields=name,cca2,ccn3,cca3,unMember,currencies,capital,maps

Endpoint 2:
https://restcountries.com/v3.1/all?fields=ccn3,cca3,region,latlng,area,flag,population,continents
```

Both tables merged in Power Query on `cca3` (ISO 3-letter code) via **Merge Queries as New → Left Outer Join**.

### API 2 — Open-Meteo (Real-Time Weather)

```
https://api.open-meteo.com/v1/forecast?latitude={lat}&longitude={lon}&current_weather=true
```

Called **once per country** via a custom Power Query function — 200+ automatic API calls at refresh.

### API 3 — FlagCDN (Country Flag Images)

```
https://flagcdn.com/160x120/{cca2_lowercase}.png
```

URL built directly in Power Query via string concatenation — no API key required.

## ⚙️ Power Query Pipeline

### Step 1 — API Ingestion

```
Home → New Source → Web → [endpoint URL]
Advanced → HTTP Request Header Parameters:
  Accept-Encoding : identity
```

> The `Accept-Encoding: identity` header is required for Open-Meteo — prevents compressed responses that Power BI cannot parse.

### Step 2 — Column Pruning & Renaming

Removed all redundant name and currency columns generated by JSON expansion. Renamed all columns to clear business labels. Verified and corrected data types.

### Step 3 — Unpivoting Nested Columns

Three column groups arrived heavily nested — one column per value, 150+ columns mostly null:

| Column group | Before | After |
|-------------|--------|-------|
| `languages` | 154 columns, mostly null | 2 columns: `cca2` + `language` |
| `continents` | Multiple columns | 2 columns: `cca2` + `continent` |
| `currencies` | Multiple columns | 2 columns: `cca2` + `currency` |

```
Select all language / continent / currency columns
→ Transform → Unpivot Only Selected Columns
```

Result: one row per country-language, country-continent, country-currency pair — three separate normalized tables.

### Step 4 — Currency Cleaning

After unpivoting, each currency produced two rows — one for the name and one for the symbol (€, $, £...). Symbols needed to be removed.

**Custom column filter:**
```m
Text.Length([Value]) > 3
and
Text.Select([Value], {"A".."Z","a".."z"," "}) = [Value]
```

- `Text.Length > 3` → removes single or short-character symbols
- `Text.Select` → keeps only alphabetic characters and spaces — removes any row containing `.`, `$`, `€`, `£`

Filter column on `TRUE` → delete helper column.

### Step 5 — Weather Data via Custom M Function

**Test call (single country):**
```
https://api.open-meteo.com/v1/forecast?latitude=12.11666666&longitude=-61.66666666&current_weather=true
```

**Transformations applied to raw JSON response:**
```
Source → Into Table
→ Select Value column → Transform → Transpose
→ Transform → Use First Row as Headers
→ Keep only: temperature, windspeed, winddirection, weathercode
```

**Converting to reusable function (`fxgetweather`):**
```
Duplicate query → Home → Advanced Editor
```

Wrap all steps in:
```m
let fxgetweather = (url_api as text) =>
    let
        Source = Web.Contents(url_api, [...]),
        ...
        -- replace hardcoded URL with url_api parameter
    in
        result
in fxgetweather
```

**Invoking the function for all countries:**

1. Add custom column `custom_url`:
```m
"https://api.open-meteo.com/v1/forecast?latitude="
& Text.From([latitude])
& "&longitude="
& Text.From([longitude])
& "&current_weather=true"
```

> `Text.From()` is required — Power Query cannot concatenate numbers with `&` directly.

2. **Add Column → Invoke Custom Function**
   - New column name: `custom_api_return`
   - Function: `fxgetweather`
   - Parameter `url_api`: column `custom_url`

> **Firewall error fix:** File → Options → Current File → Privacy → **"Ignore Privacy Levels"** — required when combining data from multiple external sources in a single query.

### Step 7 — Flag URL Column

```m
flag_url = "https://flagcdn.com/160x120/" & Text.Lower([cca2]) & ".png"
```

`Text.Lower()` is required — FlagCDN URLs are case-sensitive and only accept lowercase country codes.

## 🧩 Data Model

### Tables

| Table | Source | Key column |
|-------|--------|------------|
| `All_data` | Merged API 1 + API 2 + weather | `cca2` |
| `Languages` | Unpivoted from API 1 | `cca2` |
| `Continents` | Unpivoted from API 1 | `cca2` |
| `Currencies` | Unpivoted from API 1 | `cca2` |
| `weather_code` | Static M table | `weather_code` |
| `MetricsTable` | DAX calculated table | `metric value` |

All relationships set to **bidirectional** — required for slicer filters (Language, Continent) to propagate correctly to `All_data`.

## 📐 DAX Measures — Full Reference

### Dynamic Metric Selector Table

```dax
MetricsTable =
DATATABLE(
    "metric name", STRING,
    "metric value", STRING,
    {
        { "area (km2)",    "area"       },
        { "temperature",   "temp"       },
        { "windspeed",     "windspeed"  },
        { "population",    "population" }
    }
)
```

### Dynamic Measure — `selectedMetricValue`

```dax
selectedMetricValue =
VAR selectedMetric = SELECTEDVALUE(MetricsTable[metric value])
RETURN
    SWITCH(
        selectedMetric,
        "area",       AVERAGE(All_data[area]),
        "temp",       AVERAGE(All_data[temperature]),
        "windspeed",  AVERAGE(All_data[windspeed]),
        "population", AVERAGE(All_data[population]),
        BLANK()
    )
```

The map fill color is bound to `selectedMetricValue` — changing the slicer selection recolors the entire map instantly.

### Tooltip Concatenation Measures

```dax
Continents all  = CONCATENATEX(Continents,  Continents[continents], ", ")
Languages all   = CONCATENATEX(Languages,   Languages[languages],   ", ")
Currencies all  = CONCATENATEX(Currencies,  Currencies[Value],      ", ")
```

Required because each country has multiple languages, continents, and currencies stored as separate rows — `CONCATENATEX` reunifies them as a comma-separated string for tooltip display.

All measures are **fully filter-aware** — they adapt automatically to any active slicer on Continent, Language, or Temperature.

## 📈 Dashboard Structure

### Main Page — World Map

| Element | Configuration |
|---------|--------------|
| **Visual** | Azure Maps |
| **Fill color** | Conditional formatting → `selectedMetricValue` |
| **Map theme** | Dark (Format visual → Map Settings → Dark) |
| **Slicers** | Continent · Language · Temperature range · Metric selector |
| **Tooltip** | Custom tooltip page linked via Format visual → General → Tooltips |

### Custom Tooltip Page — Country Profile

**Dimensions:** 500 × 240 px
**Configuration:** Visualizations → Format your report page → Canvas settings → Type: Custom → Page Information → Allow use as tooltip ✅

| Element | Content | Visual type |
|---------|---------|-------------|
| Country name | Text label | Text box |
| Flag | `flag_url` column | **Simple Image** (custom visual) |
| Capital | KPI Card | Card |
| Population | KPI Card | Card |
| Area (km²) | KPI Card | Card |
| Continent(s) | `Continents all` measure | Card |
| Language(s) | `Languages all` measure | Card |
| Currency(ies) | `Currencies all` measure | Card |
| Temperature | KPI Card | Card |
| Wind speed | KPI Card | Card |
| Wind direction | KPI Card | Card |
| Weather icon | `IconURL` from `weather_code` table | **Simple Image** (custom visual) |

> **Simple Image** is a third-party custom visual downloaded from AppSource — it takes an image URL as input and renders it inline. Required for flag and weather icon display since Power BI has no native image-from-URL visual.
